## Motivation

SEO folklore often claims tools outperform blogs for links and brand search.
This experiment asks a narrower AI-visibility question: **entity association
strength** in LLM answers.

### Facts

- Tools and articles are different content genres with different user behaviors.
- Both can target the same keyword/topic cluster.
- Markup candidates: [`SoftwareApplication`](https://schema.org/SoftwareApplication) for tools, [`Article`](https://schema.org/Article) for posts; validate via [Schema Markup Validator](https://validator.schema.org/).
- Publisher context: [InfoWebPlus](https://infowebplus.com/), author [George Barbu](https://barbu.es/about/).

### Assumptions

- AI systems may mention tools when users ask “how do I …?” more often than
  they mention blog posts.
- Association strength can be proxied by brand mention rate in solution-oriented
  prompts.

### Unknowns

- Whether any lift is due to backlinks/authorship graph rather than the tool UX.
- Longevity of associations if the tool is rarely updated.

---

## Hypothesis

**H₀:** Brand mention / correct attribution rates do not differ between the
tool URL condition and the blog URL condition for matched topics.

**H₁:** The free tool condition yields higher correct brand↔capability
association on task-oriented prompts than the blog condition.

---

## Methodology

### Design type

Two-arm content deploy on similar site authority:

- Arm T: free tool (interactive or downloadable utility) + minimal docs
- Arm B: blog article covering the same problem without a tool

Prefer parallel publish dates. If sequential, document order effects.

## Variables

### Independent variables

- Content type: free tool vs blog article

### Dependent variables

- Brand mention rate on “how do I solve [PROBLEM]?” prompts
- Correct tool attribution rate (“built by / offered by”)
- URL citation rate (tool vs blog)
- Instructional accuracy (does the answer actually solve the task?)

### Controlled / held constant

- Topic / problem statement
- Author byline where applicable (coordinate with Exp-008)
- Promotion budget (ideally zero paid; if not, equalize and disclose)

### Confounds to monitor

- Organic backlinks preferentially to the tool
- App-store or directory listings
- Users pasting tool output into chats (contamination)

---

## Procedure

1. Select a problem with clear success criteria and low legal/PII risk.
2. Ship Arm T and Arm B; record URLs and publish timestamps.
3. Freeze prompt suite mixing branded and unbranded problem queries.
4. Probe weekly; score association and citation.
5. Optionally track human usage metrics as covariates, not primary outcomes.

### Prompt protocol

```text
How can I [SOLVE_PROBLEM] for free?
What tools exist to [SOLVE_PROBLEM]?
Is there a tool by [ORGANIZATION] for [SOLVE_PROBLEM]?
Summarize how to [SOLVE_PROBLEM] and cite useful pages.
```

## Limitations

- “Free tool” quality variance can dominate genre effects.
